# 2.5 — Fase 4: CIERRE (Close)

## Objetivo

Coordinar aceptación formal de entregables, informar sobre desempeño, registrar lecciones aprendidas, cerrar administrativamente el proyecto.

---

## Flujo de la Fase de Cierre

```
┌─────────────────────────────────────────────────────────────────┐
│                    FASE 4: CIERRE                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. REUNIÓN DE REVISIÓN DE FIN DE PROYECTO                      │
│     └─ Evaluar resultados con CDP y stakeholders               │
│                                                                  │
│  2. CAPTURA DE LECCIONES APRENDIDAS                             │
│     └─ Documentar éxitos, fracasos, mejoras                    │
│                                                                  │
│  3. INFORME DE FIN DE PROYECTO                                  │
│     └─ Formalizar cierre y resultados                           │
│                                                                  │
│  4. CIERRE ADMINISTRATIVO                                       │
│     └─ Archivar documentos, liberar recursos, cerrar contratos  │
│                                                                  │
│  5. TRANSICIÓN A OPERACIONES                                    │
│     └─ Entregar a equipo de operación/mantenimiento            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Actividades Detalladas

### 1. Reunión de Revisión de Fin de Proyecto

**Participantes:** CDP completo, DP, RN, representantes del EDP, PS

**Objetivos:**
- Revisar entregables finales
- Evaluar desempeño contra objetivos
- Identificar éxitos y áreas de mejora
- Aprobar cierre formal

---

### 2. Captura de Lecciones Aprendidas

**Responsable:** Director de Proyecto (DP)

**Áreas a documentar:**

| Área | Preguntas |
|------|-----------|
| **Gestión del alcance** | ¿El alcance fue bien definido? ¿Hubo desviaciones? |
| **Gestión del tiempo** | ¿El cronograma fue realista? ¿Hubo retrasos? |
| **Gestión de costos** | ¿El presupuesto fue suficiente? ¿Hubo sobrecostos? |
| **Gestión de calidad** | ¿Los entregables cumplieron expectativas? |
| **Gestión de comunicación** | ¿La comunicación fue efectiva? |
| **Gestión de riesgos** | ¿Los riesgos fueron bien gestionados? |
| **Gestión de stakeholders** | ¿Las partes interesadas estuvieron satisfechas? |
| **Gestión del equipo** | ¿El equipo funcionó bien? ¿Hubo conflictos? |
| **Herramientas y procesos** | ¿Las herramientas fueron útiles? ¿Qué se puede mejorar? |

**Formato:**
- Qué fue bien (mantener)
- Qué no fue bien (evitar)
- Recomendaciones para futuros proyectos

---

### 3. Informe de Fin de Proyecto

**Propósito:** Documentar formalmente la conclusión del proyecto

**Secciones principales:**

| Sección | Contenido |
|---------|-----------|
| **1. Resumen ejecutivo** | Estado final, resultado principal |
| **2. Objetivos** | Grado de cumplimiento de objetivos |
| **3. Alcance** | Qué se entregó vs. planeado |
| **4. Cronograma** | Duración real vs. planificada |
| **5. Costos** | Inversión real vs. presupuesto |
| **6. Calidad** | Cumplimiento de criterios de calidad |
| **7. Riesgos** | Riesgos ocurridos y cómo se manejaron |
| **8. Lecciones aprendidas** | Resumen de lecciones |
| **9. Recomendaciones** | Para futuros proyectos o operación |
| **10. Anexos** | Documentos de soporte |

**Artefacto:** `OPM2-33.C.TPL` — Informe de Fin de Proyecto

---

### 4. Cierre Administrativo

**Tareas:**

| Tarea | Descripción |
|-------|-------------|
| **Archivar documentos** | Organizar y almacenar toda la documentación del proyecto |
| **Liberar recursos** | Devolver recursos humanos, equipos, espacios |
| **Cerrar contratos** | Formalizar cierre de contratos con proveedores |
| **Cerrar cuentas** | Cerrar cuentas de proyecto, accesos temporales |
| **Actualizar registros** | Registrar proyecto como cerrado en sistemas de gestión |
| **Distribuir activos** | Reasignar equipos, licencias, materiales sobrantes |

---

### 5. Transición a Operaciones

**Responsable:** Responsable de Negocio (RN) + Grupo de Implementación en el Negocio (GIN)

**Actividades:**
- Transferir conocimiento al equipo de operación
- Entregar documentación de operación
- Realizar formación a usuarios finales
- Establecer acuerdos de soporte
- Definir métricas de operación

## Duración y Esfuerzo

| Perfil de Proyecto | Duración típica | Esfuerzo |
|-------------------|-----------------|----------|
| **Light** | 2-3 días | 16-24 horas |
| **Medium** | 1-2 semanas | 40-80 horas |
| **Heavy** | 2-4 semanas | 80-160 horas |

---

## Checklist de Cierre

- [ ] Todos los entregables aceptados formalmente
- [ ] Informe de fin de proyecto completo
- [ ] Lecciones aprendidas documentadas y compartidas
- [ ] Documentación archivada
- [ ] Recursos liberados
- [ ] Contratos cerrados
- [ ] Transición a operaciones completada
- [ ] Reunión de cierre realizada
- [ ] Aprobación formal de cierre por CDP
- [ ] Proyecto marcado como cerrado en registros

## Post-Cierre

### Evaluación Post-Proyecto (3-6 meses después)

| Aspecto | Evaluación |
|---------|------------|
| **Beneficios realizados** | ¿Se lograron los beneficios esperados? |
| **Sostenibilidad** | ¿La solución sigue funcionando? |
| **Impacto organizacional** | ¿Hubo impactos no previstos? |
| **ROI** | ¿Se obtuvo retorno de la inversión? |
