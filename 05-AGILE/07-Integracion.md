# 5.7 — Integración PM² + Ágil

## Cuándo Usar Cada Enfoque

| Situación | Recomendación |
|-----------|---------------|
| **Consultoría institucional** con requisitos definidos | PM² Tradicional (Medium/Heavy) |
| **Desarrollo de producto tecnológico** | PM² Agile (Medium) |
| **Proyecto híbrido: gobernanza + innovación** | PM² Híbrido: Fases 1-2 tradicional + Ejecución ágil |
| **Proyecto público con entregables regulatorios** | PM² Tradicional con elementos ágiles |
| **Startup/Experimentación** | PM² Agile (Light) |

---

## Modelo Híbrido Recomendado para Innovadataco

```
┌──────────────────────────────────────────────────────────────────────┐
│                    PROYECTO HÍBRIDO INNOVADATACO                    │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  FASE 1: INICIO                                                      │
│  ├── PM² Tradicional (completo)                                      │
│  ├── Caso de negocio formal                                          │
│  ├── Acta de constitución                                            │
│  └── LpP formal                                                      │
│                                                                      │
│  FASE 2: PLANIFICACIÓN                                               │
│  ├── PM² Tradicional (alto nivel)                                    │
│  ├── Manual del proyecto                                             │
│  ├── Arquitectura de releases                                        │
│  ├── Roadmap de producto                                             │
│  └── LpE formal                                                      │
│                                                                      │
│  FASE 3: EJECUCIÓN                                                   │
│  ├── PM² Agile (iteraciones)                                         │
│  ├── Sprints de 2 semanas                                            │
│  ├── Releases cada 2-4 sprints                                       │
│  ├── Demo a stakeholders cada sprint                                   │
│  └── Retrospectivas continuas                                        │
│                                                                      │
│  FASE 4: CIERRE                                                      │
│  ├── PM² Tradicional (completo)                                      │
│  ├── Aceptación formal de entregables                                │
│  ├── Informe de fin de proyecto                                        │
│  └── Transición a operaciones                                        │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

## Gestión de Stakeholders en Proyectos Ágiles

### Participación por Rol

| Rol | Participación Ágil |
|-----|-------------------|
| **PP (Product Owner)** | Refina backlog, asiste a reviews, prioriza |
| **RN (Stakeholder Rep)** | Proporciona feedback, participa en reviews |
| **DP (Scrum Master)** | Facilita ceremonias, remueve impedimentos |
| **PS (Technical Advisor)** | Revisa arquitectura, asiste a planning |

### Frecuencia de Involucramiento

| Stakeholder | Frecuencia | Actividad |
|-------------|-----------|-----------|
| Usuarios finales | Cada sprint (review) | Validar incrementos |
| Dirección | Cada 2-4 sprints | Revisar avance estratégico |
| Equipo técnico | Diaria (stand-up) | Sincronización |
| Cliente (PP) | Semanal (refinamiento) | Priorizar, clarificar |

## Artefactos en Proyectos Híbridos

| Fase | Artefactos PM² Tradicional | Artefactos Ágiles |
|------|---------------------------|-------------------|
| **Inicio** | Caso de negocio, Acta | — |
| **Planificación** | Manual del proyecto | Product Backlog, Roadmap |
| **Ejecución** | Informes de progreso | Sprint Backlog, Burndown, Incrementos |
| **Cierre** | Informe de fin, Lecciones | Retrospectivas documentadas |

---

## Transición entre Enfoques

| Transición | Recomendación |
|------------|---------------|
| Tradicional → Ágil | Definir visión y alcance alto nivel antes de empezar sprints |
| Ágil → Tradicional | Congelar alcance, completar planificación detallada |
| Híbrido | Mantener gobernanza PM² (CDP, fases) + ejecución ágil |
