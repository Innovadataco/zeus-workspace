# FICHA TÉCNICA CC-13 V3
## Sistema de Percepción y Detección (SPD)
### Speed Detection System

---

## 1. RESUMEN EJECUTIVO

El **Sistema de Percepción y Detección (SPD)** es una solución integral de control de velocidad vehicular que combina tecnologías de radar Doppler, cámaras de reconocimiento automático de placas (ANPR) y paneles de mensaje variable (PMV) para el monitoreo, detección y notificación en tiempo real del comportamiento de velocidad de los vehículos en corredores viales estratégicos.

Este sistema permite implementar controles de velocidad promedio por tramo, detectar excesos de velocidad instantáneos, identificar vehículos infractores y comunicar alertas visuales inmediatas, contribuyendo significativamente a la reducción de la siniestralidad vial y al fomento de la cultura de cumplimiento normativo.

**Valor estratégico para el corredor:** El SPD transforma el control de velocidad de una medida puntual y reactiva a un sistema proactivo y preventivo que modifica el comportamiento del conductor en tiempo real, generando conciencia vial instantánea y datos estadísticos para la toma de decisiones de planeación vial.

## 3. DESCRIPCIÓN FUNCIONAL/TÉCNICA

### 3.1 Propósito del Sistema

El SPD tiene como objetivo principal monitorear, detectar y registrar el comportamiento de velocidad de los vehículos que transitan por el corredor vial, implementando mecanismos de control tanto preventivos como sancionatorios mediante la integración de múltiples tecnologías de detección.

### 3.3 Componentes Principales

| Componente | Función |
|------------|---------|
| **Radares Doppler** | Detección de velocidad instantánea y cálculo de velocidad promedio por tramo |
| **Cámaras ANPR** | Captura y reconocimiento óptico de caracteres de placas vehiculares |
| **Controlador Local** | Procesamiento en tiempo real, emparejamiento de vehículos entre puntos |
| **Paneles de Mensaje Variable** | Visualización de alertas, velocidades detectadas y mensajes preventivos |
| **Sistema SIRI** | Interfaz para gestión de sanciones y notificaciones a infractores |
| **Centro CCO** | Monitoreo centralizado, reportes estadísticos y gestión de alertas |

### 3.4 Flujo de Operación

1. **Detección de Entrada:** El vehículo es detectado por el radar/cámara ANPR en el punto de control inicial
2. **Registro de Datos:** Se captura velocidad, placa, timestamp, clase vehicular y dirección
3. **Detección de Salida:** El mismo vehículo es detectado en el punto de control final
4. **Cálculo:** El sistema calcula velocidad promedio del tramo y verifica excesos instantáneos
5. **Acción:** Según configuración, se activa PMV, se genera alerta CCO o se transfiere a SIRI

## 5. REQUERIMIENTOS FUNCIONALES

### RF-01: Detección de Velocidad Instantánea
El sistema debe detectar y registrar la velocidad instantánea de cada vehículo que transita por el punto de control con precisión de ±1 km/h.

### RF-02: Cálculo de Velocidad Promedio por Tramo
El sistema debe calcular la velocidad promedio de cada vehículo entre dos puntos de control distantes entre 500m y 10km, utilizando el emparejamiento de placas.

### RF-03: Reconocimiento Automático de Placas (ANPR)
El sistema debe capturar imágenes de alta resolución y realizar OCR para identificar la placa del vehículo en condiciones diurnas y nocturnas.

### RF-04: Clasificación Vehicular Automática
El sistema debe clasificar automáticamente los vehículos detectados en categorías: motocicletas, vehículos ligeros, vehículos pesados y buses.

### RF-05: Integración con Paneles de Mensaje Variable
El sistema debe enviar mensajes automáticos a los PMV para mostrar: velocidad detectada, alertas de exceso, mensajes preventivos y tiempos de viaje estimados.

### RF-06: Generación de Alertas en Tiempo Real
El sistema debe generar alertas instantáneas cuando se detecten excesos de velocidad configurables, enviando notificaciones al CCO y a operadores viales.

### RF-07: Almacenamiento de Evidencias
El sistema debe almacenar fotografías, videos cortos y datos de detección de cada evento de velocidad por mínimo 30 días con posibilidad de exportación.

### RF-08: Interfaz con Sistema SIRI
El sistema debe transferir automáticamente los datos de infracciones de velocidad al Sistema Integral de Registro de Infracciones para su gestión sancionatoria.

### RF-09: Reportes Estadísticos
El sistema debe generar reportes automáticos y bajo demanda con: velocidades promedio, percentiles 85, conteos vehiculares, horas pico y mapas de calor de velocidad.

### RF-10: Operación Remota y Mantenimiento
El sistema debe permitir la configuración remota, actualización de firmware, diagnóstico de salud del equipo y gestión de parámetros sin visita física al sitio.

## 9. CASOS DE USO

### CU-01: Control de Velocidad Promedio en Tramo
**Actor:** Conductor de vehículo particular
**Descripción:** El sistema calcula la velocidad promedio del vehículo entre dos puntos de control separados 2 km. Si el promedio supera el límite, se genera la infracción.
**Flujo:** El conductor pasa por el punto A (detección + ANPR) → circula por el tramo → pasa por el punto B (detección + ANPR) → sistema calcula velocidad promedio → si hay exceso, transfiere a SIRI.
**Valor:** Mide el comportamiento real del conductor durante todo el trayecto, no solo en el punto de control.
**Referencia:** Implementación en túneles de la Autopista Norte de Bogotá.

### CU-02: Detección y Alerta de Exceso Instantáneo
**Actor:** Operador de tránsito CCO
**Descripción:** Un vehículo circula a 95 km/h en una zona de 60 km/h. El radar detecta el exceso, la cámara captura la placa y el PMV muestra "VELOCIDAD: 95 km/h - REDUZCA".
**Flujo:** Radar detecta velocidad > límite → Cámara captura placa → PMV activa mensaje de alerta → Operador CCO recibe notificación → Decisión de acción preventiva.
**Valor:** Feedback inmediato al conductor que modifica comportamiento en el momento.
**Referencia:** Corredores viales de la ciudad de Medellín con PMV integrados.

### CU-03: Identificación de Vehículos de Interés
**Actor:** Autoridad de tránsito / Policía
**Descripción:** El sistema detecta un vehículo reportado por alerta de seguridad o con restricciones de circulación.
**Flujo:** Vehículo pasa por control → ANPR identifica placa → Consulta en lista de alertas → Coincidencia detectada → Alerta prioritaria a CCO y autoridades.
**Valor:** El sistema SPD puede integrarse con listas de vehículos buscados o con restricciones de pico y placa.
**Referencia:** Sistemas de vigilancia en corredores viales de Cali.

### CU-04: Análisis de Patrones de Velocidad para Planeación Vial
**Actor:** Ingeniero de tránsito / Planeador urbano
**Descripción:** El sistema acumula datos de velocidad durante 6 meses para analizar comportamiento por franjas horarias y días de la semana.
**Flujo:** Recolección continua de datos → Generación de reportes estadísticos → Identificación de patrones → Propuestas de ajuste de límites de velocidad → Implementación de medidas.
**Valor:** Datos objetivos para decisiones de ingeniería de tránsito basada en evidencia.
**Referencia:** Estudios de velocidad en corredores de Bogotá utilizados por el IDU.

### CU-05: Respuesta a Incidentes con Evidencia Documentada
**Actor:** Investigador de siniestros / Aseguradora
**Descripción:** Tras un accidente en el corredor, se consultan los registros del SPD para determinar velocidades previas del vehículo involucrado.
**Flujo:** Solicitud de información → Consulta en base de datos por timestamp y ubicación → Recuperación de imágenes y datos de velocidad → Entrega de evidencia certificada.
**Valor:** Evidencia técnica para investigaciones de siniestros y procesos judiciales.
**Referencia:** Utilización de registros fotomulta en investigaciones de accidentes en autopistas concesionadas.
