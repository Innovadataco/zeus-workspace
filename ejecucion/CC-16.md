# 8.1 CC-16: EVCS (Electric Vehicle Charging Station)

## 8.1.1 Resumen Ejecutivo

El componente **CC-16 EVCS** constituye la infraestructura de recarga eléctrica para vehículos del corredor vial Chía–Girardot, proporcionando puntos de carga estratégicamente ubicados que habilitan la movilidad eléctrica a lo largo de los 306 km del corredor. Este subsistema es fundamental para la transición energética del transporte y la descarbonización del sector vial en Colombia.

La electrificación del transporte representa un pilar de la política de movilidad sostenible. El sistema EVCS no solo proporciona energía para vehículos eléctricos (EV), sino que se integra con la gestión energética del corredor (CC-39 MSG, CC-41 E-LOG) y puede actuar como almacenamiento distribuido para servicios auxiliares de red (V2G - Vehicle to Grid).

El componente EVCS se despliega en zonas de servicio, peajes y áreas de descanso del corredor, con una densidad que garantiza autonomía suficiente para recorrer el corredor completo en vehículos eléctricos de gama media. La infraestructura contempla múltiples estándares de carga y conectores para compatibilidad con la diversidad del parque automotor.

## 8.1.3 Descripción Funcional / Técnica

### Alcance de la Solución

El sistema EVCS comprende estaciones de recarga de diferentes niveles de potencia distribuidas estratégicamente:

- **Carga rápida (DC Fast Charging)**: 50-350 kW para recarga en 15-30 minutos
- **Carga semi-rápida (AC/DC)**: 22-50 kW para recarga en 1-2 horas
- **Carga lenta (AC)**: 7-22 kW para estacionamiento prolongado

Cada estación EVCS incluye:
- Puntos de carga con múltiples conectores (CCS, CHAdeMO, GB/T, Tipo 2)
- Sistema de gestión de carga (CMS) para control de acceso y facturación
- Integración con sistema de pago (tarjetas, apps, contactless)
- Pantalla informativa para usuarios
- Protecciones eléctricas y sistema de seguridad
- Monitoreo remoto de estado y consumo
- Infraestructura eléctrica de media y baja tensión

Las estaciones se conectan al CCO para monitoreo de disponibilidad, estado operativo, y gestión de emergencias, integrándose también con aplicaciones móviles de usuarios (CC-26 MOB) para información de disponibilidad en tiempo real.

### Justificación Estratégica

La movilidad eléctrica en Colombia está en crecimiento acelerado, impulsada por:
- Política nacional de descarbonización y compromisos de COP26
- Beneficios arancelarios y tributarios para EV
- Crecimiento de la red de distribución de energía renovable
- Preocupación creciente por calidad del aire en ciudades

Sin infraestructura de carga en corredores interurbanos, los vehículos eléctricos quedan confinados a uso urbano. El EVCS habilita:
- **Viajes interurbanos**: Recorridos de larga distancia sin "range anxiety"
- **Turismo sostenible**: Atracción de turismo con vehículos eléctricos
- **Electrificación de flotas**: Habilita operación de buses y flotas comerciales
- **Resiliencia energética**: Almacenamiento distribuido ante emergencias (V2G)

## 8.1.4 Especificación Técnica

| Parámetro | Especificación Mínima |
|-----------|----------------------|
| **Potencia DC rápida** | ≥ 150 kW por punto (capacidad de upgrade a 350 kW) |
| **Potencia AC** | 22 kW trifásico / 7.4 kW monofásico |
| **Conectores DC** | CCS Combo 2 (estándar), CHAdeMO (compatible) |
| **Conectores AC** | Tipo 2 (Mennekes) |
| **Eficiencia del cargador** | ≥ 95% desde AC entrada a DC salida |
| **Factor de potencia** | ≥ 0.95 |
| **THD (distorsión armónica)** | < 5% cumpliendo IEEE 519 |
| **Protección eléctrica** | Sobretensión, subtensión, cortocircuito, protección diferencial |
| **Grado de protección** | IP54 para electrónica, IP65 para conectores |
| **Rango térmico** | -10°C a +50°C operación, almacenamiento -20°C a +60°C |
| **Comunicación** | OCPP 2.0.1, Ethernet, 4G/5G, WiFi |
| **Interfaz usuario** | Pantalla táctil ≥ 7", indicadores LED, audio |
| **Sistema de pago** | Tarjeta RFID, QR, app móvil, tarjeta de crédito contactless |
| **Cable de carga** | Longitud ≥ 5 m, gestión automática de cable |
| **Tiempo de recarga** | 10-80% SOC en < 30 min (150 kW, batería 60 kWh) |
| **Respaldo energético** | Conexión a CC-39 MSG para operación con micro-red |

## 8.1.5 Requerimientos Funcionales (RF)

| ID | Nombre | Descripción Funcional / Técnica | Justificación Estratégica |
|----|--------|--------------------------------|---------------------------|
| **RF-01** | Múltiples Estándares de Conector | Cada punto de carga rápida debe incluir CCS Combo 2 y opcionalmente CHAdeMO. Los puntos AC deben usar Tipo 2. El sistema debe detectar automáticamente el tipo de conector y vehículo conectado. | La diversidad del parque automotor eléctrico requiere compatibilidad con múltiples estándares. CCS es el estándar europeo/internacional adoptado en Colombia, pero CHAdeMO aún tiene presencia en vehículos asiáticos. |
| **RF-02** | Gestión de Carga Inteligente | El sistema debe gestionar la potencia disponible entre múltiples puntos de carga simultáneos, priorizando según: tiempo de espera, nivel de batería del vehículo, plan de carga del usuario, y demanda de red. Debe soportar "load balancing" dinámico. | La potencia eléctrica disponible en ubicaciones remotas puede ser limitada. La gestión inteligente maximiza el uso de infraestructura existente sin requerir upgrades costosos de transformadores. |
| **RF-03** | Autenticación y Acceso | Los usuarios deben poder autenticarse mediante: tarjeta RFID (propia o interoperable), código QR desde app móvil, o pago con tarjeta de crédito sin registro previo. El sistema debe soportar roaming con otras redes de carga. | La facilidad de acceso es crítica para adopción. Los usuarios ocasionales (turistas) no tendrán tarjetas específicas del sistema y requieren opción de pago directo. El roaming habilita interoperabilidad nacional. |
| **RF-04** | Información al Usuario | La pantalla debe mostrar: estado de conexión, potencia de carga actual, energía entregada (kWh), costo acumulado, tiempo transcurrido, tiempo estimado de finalización, y mensajes de error/desconexión. Debe soportar múltiples idiomas (español, inglés). | La transparencia en información genera confianza. Los usuarios necesitan saber cuánto les costará la carga y cuánto tiempo tomará para planificar su viaje. |
| **RF-05** | Monitoreo Remoto y Mantenimiento Predictivo | El sistema debe reportar al CMS: estado operativo, energía total entregada, número de sesiones, fallos detectados, temperatura interna, y estado de conectores/cables. Debe predecir necesidades de mantenimiento basado en uso y condiciones. | El monitoreo remoto permite detección proactiva de fallos antes de que afecten usuarios. La predicción de mantenimiento optimiza costos operativos y disponibilidad del servicio. |
| **RF-06** | Integración con Red Eléctrica Inteligente | El sistema debe comunicarse con CC-39 MSG para: adaptar carga según disponibilidad de generación renovable, participar en programas de respuesta a demanda, y soportar V2G (Vehicle to Grid) para inyección de energía a la red cuando sea técnicamente viable. | La integración con micro-redes inteligentes maximiza uso de energía renovable y reduce costos energéticos. El V2G transforma los EV en almacenamiento distribuido para servicios auxiliares. |
| **RF-07** | Seguridad Eléctrica y Protección | El sistema debe incluir protección contra: sobretensiones (transitorios y permanentes), subtensiones, cortocircuitos, corrientes de fuga (protección diferencial tipo B para DC), sobrecargas, y fallas de aislamiento. Debe cumplir normativa eléctrica vigente. | La seguridad eléctrica es primordial en equipos de alta potencia manipulados por usuarios no técnicos. Las fallas pueden causar electrocución, incendios o daños a vehículos costosos. |
| **RF-08** | Reserva y Programación de Carga | Los usuarios deben poder reservar puntos de carga mediante app móvil (CC-26) con antelación de hasta 24 horas. El sistema debe soportar programación de carga diferida (cargar durante horas valle tarifarias) y límites de carga (ej: cargar hasta 80%). | La reserva elimina incertidumbre en viajes planificados. La carga programada aprovecha tarifas más bajas y reduce carga en horas pico de demanda eléctrica. |
| **RF-09** | Gestión de Emergencias | Ante emergencias (incendio vehículo, accidente, falla eléctrica grave), el sistema debe poder desconectar remotamente todos los puntos de carga, activar alarmas locales, notificar al CCO, y coordinar con servicios de emergencia. Debe incluir parada de emergencia física. | La rápida desconexión de energía es crítica ante incidentes. Un vehículo eléctrico en llamas conectado a cargador puede propagar incendio o electrocutar rescatistas. |
| **RF-10** | Integración con ITS y Navegación | El estado de disponibilidad de cargadores debe integrarse con: app móvil del corredor (CC-26), sistemas de navegación (Waze, Google Maps), y paneles de mensajería variable (CC-03). Debe actualizar disponibilidad en tiempo real (< 2 minutos). | La integración con navegación permite a conductores planificar paradas de recarga. La información de disponibilidad en tiempo real evita llegar a estaciones saturadas. |

## 8.1.6 Requisitos No Funcionales (RNF)

| ID | Nombre | Descripción | Justificación |
|----|--------|-------------|---------------|
| **RNF-01** | Disponibilidad del Servicio | El sistema EVCS debe mantener disponibilidad ≥ 98% para puntos de carga rápida, medida mensualmente por estación. La indisponibilidad se define como imposibilidad de iniciar una sesión de carga. | Los usuarios dependen de la carga para completar su viaje. Un punto de carga fuera de servicio puede dejar varados a usuarios sin alternativas cercanas. |
| **RNF-02** | Tiempo de Inicio de Carga | El tiempo desde autenticación hasta inicio de entrega de energía debe ser < 30 segundos para el 95% de las sesiones. El sistema debe detectar y rechazar conexiones impropias inmediatamente. | Los usuarios esperan experiencia similar a cargar combustible (< 5 minutos para autenticación + carga parcial). Retardos excesivos generan insatisfacción y congestión en estaciones. |
| **RNF-03** | Eficiencia Energética | El sistema debe operar con eficiencia ≥ 95% desde punto de conexión a red hasta vehículo. Las pérdidas deben disiparse sin comprometer equipos adyacentes o seguridad de usuarios. | La ineficiencia se traduce en costos operativos y calor disipado que puede afectar equipos y requerir refrigeración adicional. |
| **RNF-04** | Ciberseguridad y Protección de Datos | Las comunicaciones deben cifrarse (TLS 1.3). Los datos de usuarios y transacciones deben protegerse según Ley 1581 de 2012. El acceso remoto de mantenimiento requiere autenticación multifactor. Los cargadores deben resistir ataques de negación de servicio. | Los cargadores son objetivos de ciberataques que pueden dañar vehículos, robar datos de pago, o interrumpir servicio. La protección de datos de usuarios es obligatoria legalmente. |
| **RNF-05** | Resistencia Ambiental | Los equipos deben operar bajo lluvia intensa, alta humedad, polvo, y radiación solar directa. Deben resistir vandalismo (grado IK08) y manipulación no autorizada. Los conectores deben soportar ciclos de conexión ≥ 10,000. | Las estaciones están expuestas a condiciones ambientales severas y riesgo de vandalismo. Los conectores de alta potencia sufren desgaste por uso frecuente y deben ser durables. |
| **RNF-06** | Ciclo de Vida y Obsolescencia | Los componentes principales (cargadores, sistemas de gestión) deben tener ciclo de vida ≥ 10 años con soporte técnico garantizado. El diseño debe permitir upgrades de potencia sin reemplazo completo. | La infraestructura de carga representa inversión significativa. Los estándares de carga evolucionan y el sistema debe poder adaptarse sin obsolescencia prematura. |
| **RNF-07** | Accesibilidad Universal | Las estaciones deben cumplir normativa de accesibilidad: espacio para maniobra de sillas de ruedas, altura de pantalla accesible, señalización en braille, y asistencia remota disponible. | La movilidad eléctrica debe ser accesible para todos. El cumplimiento de normativa de accesibilidad es obligatorio para infraestructura pública. |

### Normativa Aplicable
- **IEC 61851**: Sistemas de carga para vehículos eléctricos
- **SAE J1772**: Conector estándar Norteamérica (referencia)
- **IEC 62196**: Conectores para carga de EV
- **OCPP 2.0.1**: Open Charge Point Protocol
- **IEEE 519**: Calidad de potencia eléctrica
- **Resoluciones CREG**: Tarifas y regulación eléctrica Colombia
- **ISO 15118**: Comunicación V2G

### Consideraciones de Instalación
- Distancia entre estaciones: máximo 100 km para garantizar autonomía
- Transformadores dedicados con capacidad ≥ 150% de demanda pico
- Sistema de enfriamiento para cargadores de alta potencia
- Accesibilidad para grúas de mantenimiento
- Señalización vial anticipada (500 m antes de la estación)
