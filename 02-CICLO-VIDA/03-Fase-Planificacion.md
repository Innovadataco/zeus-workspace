# 2.3 — Fase 2: PLANIFICACIÓN (Plan)

## Objetivo

Asignar el equipo de desarrollo, desarrollar el alcance detallado, planificar el trabajo, establecer cómo se gestionará cada área del proyecto.

---


## Actividades Detalladas

### 1. Reunión Inicial de Planificación

**Participantes:** Director de Proyecto (DP), Responsable de Negocio (RN), miembros clave del futuro EDP

**Objetivos:**
- Presentar el acta de constitución al equipo
- Definir enfoque de planificación
- Asignar responsabilidades de planificación
- Establecer calendario de planificación

---

### 2. Manual del Proyecto (Project Handbook)

**Propósito:** Documentar la estrategia global del proyecto

**Secciones principales:**

| Sección | Contenido |
|---------|-----------|
| **1. Contexto del proyecto** | Antecedentes, objetivos, caso de negocio |
| **2. Alcance del proyecto** | Descripción detallada, límites, supuestos |
| **3. Estrategia del proyecto** | Enfoque, metodología, herramientas |
| **4. Organización** | Estructura del equipo, roles, RASCI |
| **5. Gestión del proyecto** | Resumen de planes de gestión |
| **6. Calendario** | Hitos principales, cronograma de alto nivel |
| **7. Presupuesto** | Resumen de costos |
| **8. Riesgos principales** | Resumen del registro de riesgos |
| **9. Criterios de éxito** | KPIs y métricas |
| **10. Gobierno** | CDP, reuniones, informes |

**Artefacto:** `OPM2-04.P.TPL` — Manual del Proyecto

---

### 3. Planes de Gestión

PM² define planes específicos para cada área de gestión:

| Plan | Propósito | Artefacto |
|------|-----------|-----------|
| **Plan de Gestión de Requisitos** | Cómo se identificarán, documentarán y gestionarán los requisitos | OPM2-11 |
| **Plan de Gestión de Cambios** | Proceso para solicitar, evaluar y aprobar cambios | OPM2-12 |
| **Plan de Gestión de Riesgos** | Cómo se identificarán, evaluarán y mitigarán riesgos | OPM2-13 |
| **Plan de Gestión de Incidencias** | Cómo se reportarán y resolverán incidencias | OPM2-14 |
| **Plan de Gestión de Calidad** | Estándares, criterios de aceptación, QA | OPM2-15 |
| **Plan de Gestión de la Comunicación** | Audiencias, canales, frecuencia, formato | OPM2-16 |
| **Plan de Aceptación de Entregables** | Criterios, procedimientos, responsables | OPM2-08 |
| **Plan de Transición** | Cómo se pasará de proyecto a operaciones | OPM2-09 |
| **Plan de Implementación en el Negocio** | Despliegue, formación, soporte | OPM2-10 |
| **Plan de Externalización** | Gestión de contratistas y terceros | OPM2-07 |

---

### 4. Matriz de Partes Interesadas

**Propósito:** Identificar, analizar y gestionar las expectativas de todas las partes interesadas

**Contenido:**

| Columna | Descripción |
|---------|-------------|
| Nombre/Rol | Identificación de la parte interesada |
| Interés | Nivel de interés en el proyecto (Alto/Medio/Bajo) |
| Influencia | Capacidad de afectar el proyecto (Alto/Medio/Bajo) |
| Expectativas | Qué espera del proyecto |
| Impacto | Cómo el proyecto le afecta |
| Estrategia | Cómo se gestionará (involucrar, informar, monitorear) |
| Responsable PM² | Quién del equipo gestiona la relación |

**Artefacto:** `OPM2-05.P.TPL` — Matriz de Partes Interesadas

---

### 5. Plan de Trabajo del Proyecto

**Propósito:** Definir el cronograma detallado, recursos y dependencias

**Componentes:**

| Componente | Descripción |
|------------|-------------|
| **Estructura de Desglose de Trabajo (WBS)** | Jerarquía de entregables y actividades |
| **Cronograma** | Fechas de inicio/fin, hitos, ruta crítica |
| **Asignación de recursos** | Quién hace qué, cuándo |
| **Presupuesto detallado** | Costos por actividad, por fase |
| **Dependencias** | Predecesoras, sucesoras, externas |
| **Hitos** | Puntos de control clave |

**Artefacto:** `OPM2-06.P.TPL` — Plan de Trabajo del Proyecto

## Punto de Revisión: LpE (Listo para Ejecución)

### Propósito
Asegurar que el proyecto está planificado adecuadamente y el equipo está listo para comenzar el trabajo.

### Criterios de Evaluación

| Criterio | Pregunta |
|----------|----------|
| Alcance | ¿El alcance está definido claramente? |
| Cronograma | ¿El cronograma es realista y aprobado? |
| Recursos | ¿El equipo está asignado y disponible? |
| Presupuesto | ¿El presupuesto está aprobado? |
| Riesgos | ¿Los riesgos principales están identificados y mitigados? |
| Planes | ¿Todos los planes de gestión están completos? |
| Comunicación | ¿Las partes interesadas están informadas? |

### Decisiones Posibles

| Decisión | Significado |
|----------|-------------|
| **APROBAR** | Proceder a la fase de Ejecución |
| **CONDICIONAR** | Proceder con ajustes identificados |
| **RECHAZAR** | No proceder, documentar razones |
| **APLAZAR** | Posponer hasta resolver problemas |
