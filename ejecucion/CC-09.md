# CC-09: SMS / SBT - Sistema de Monitoreo de Estructuras

## 8.9.1 Resumen Ejecutivo

El Sistema de Monitoreo de Estructuras (SMS - Structural Monitoring System / SBT - Structural Bridge Technology) constituye una solución tecnológica avanzada para la **Evaluación de Salud Estructural** de puentes, viaductos, pórticos y otras obras de arte en infraestructuras de transporte. Este sistema permite monitorear tensiones en cables (torones), fatiga estructural, deformaciones y vibraciones en tiempo real, proporcionando datos críticos para la gestión del ciclo de vida de las estructuras.

La capacidad de **Evaluación de Salud Estructural** mediante el monitoreo continuo de tensiones en cables y fatiga en puentes es fundamental para garantizar la seguridad de usuarios, optimizar programas de mantenimiento predictivo y extender la vida útil de las infraestructuras. El sistema SMS/SBT integra sensores de deformación, acelerómetros, extensómetros y tecnologías de análisis estructural avanzado para proporcionar visibilidad completa del comportamiento de las estructuras bajo cargas operacionales y ambientales.

## 8.9.3 Descripción Funcional y Técnica

### 3.1 Propósito del Sistema

El SMS/SBT tiene como propósito fundamental monitorear la salud estructural de puentes y obras de arte para:

- **Detección de daños estructurales**: Identificación temprana de fisuras, corrosión y deformaciones
- **Monitoreo de tensiones en cables**: Control de fuerza en torones de puentes atirantados y colgantes
- **Evaluación de fatiga**: Análisis de ciclos de carga y vida útil remanente
- **Optimización de mantenimiento**: Transición de mantenimiento correctivo a predictivo
- **Gestión de riesgo sísmico**: Evaluación de respuesta estructural ante eventos sísmicos

## 8.9.11 Casos de Uso

### CASO DE USO 1: Monitoreo de Puente Atirantado

**Actor Principal:** Ingeniero estructural de la concesión

**Flujo Principal:**
1. El sistema mide continuamente tensión en los 48 torones del puente
2. Se detecta una disminución del 8% en la tensión de un cable
3. El sistema genera alerta de "advertencia" al ingeniero
4. El ingeniero programa inspección detallada del cable
5. Se encuentra problema en anclaje y se realiza reparación preventiva
6. Se evita falla catastrófica y se programa reemplazo programado

**Valor Generado:** Prevención de colapso estructural, optimización de mantenimiento

---

### CASO DE USO 2: Evaluación de Fatiga en Puente Vehicular

**Actor Principal:** Gestor de activos de infraestructura

**Flujo Principal:**
1. El sistema registra ciclos de carga usando datos WIM y strain gauges
2. El algoritmo Rainflow acumula daño por fatiga en elementos críticos
3. Se proyecta que la viga principal alcanzará vida útil en 7 años
4. El sistema recomienda reforzamiento estructural
5. Se programa intervención con financiamiento anticipado
6. Se extiende vida útil del puente 15 años adicionales

**Valor Generado:** Planificación financiera, extensión de vida útil

---

### CASO DE USO 3: Detección de Daño Post-Sismo

**Actor Principal:** Equipo de emergencia y respuesta sísmica

**Flujo Principal:**
1. Ocurre sismo de magnitud 6.2 con epicentro a 50 km
2. El sistema detecta aceleraciones pico superiores a umbrales
3. Se activa protocolo automático de inspección post-sismo
4. El sistema analiza cambios en frecuencias propias y deformaciones
5. Se identifica daño menor en junta de expansión
6. Se autoriza apertura con monitoreo intensivo mientras se repara

**Valor Generado:** Toma de decisiones rápida, reducción de tiempo de cierre

---

### CASO DE USO 4: Optimización de Mantenimiento Preventivo

**Actor Principal:** Director de operaciones y mantenimiento

**Flujo Principal:**
1. El sistema lleva 3 años acumulando datos de comportamiento estructural
2. El análisis muestra que puentes A y C presentan degradación acelerada
3. El puente B mantiene comportamiento estable
4. Se reasignan recursos de mantenimiento de B hacia A y C
5. Se evitan 3 inspecciones innecesarias al puente B
6. Se interviene preventivamente puentes A y C con costo 40% menor

**Valor Generado:** Optimización de recursos, mantenimiento basado en condición

---

### CASO DE USO 5: Control de Sobrecargas en Puente

**Actor Principal:** Operador de control de tráfico

**Flujo Principal:**
1. Camión con carga indivisible solicita paso por puente estructural
2. El sistema WIM detecta peso de 85 ton (límite: 60 ton)
3. Los strain gauges confirman esfuerzos cercanos al límite admisible
4. El sistema alerta sobre riesgo de sobrecarga estructural
5. Se detiene al vehículo antes del puente
6. Se redirige por ruta alterna para cargas especiales

**Valor Generado:** Protección de infraestructura, prevención de daños por sobrecarga
