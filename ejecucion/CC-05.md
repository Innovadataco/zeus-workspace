# CC-05: VDS / TDS - Sistema de Contadores de Tráfico

## 8.5.1 Resumen Ejecutivo

El Sistema de Contadores de Tráfico, conocido técnicamente como VDS (Vehicle Detection System) o TDS (Traffic Data System), constituye el componente fundamental para la **Caracterización Dinámica del Flujo Vehicular** en infraestructuras de transporte inteligente (ITS). Este sistema proporciona datos en tiempo real sobre el volumen, velocidad, clasificación y densidad del tráfico, permitiendo a los operadores tomar decisiones informadas para la gestión operativa de la vía.

La capacidad de **Cuantificar y Clasificar el Volumen Vehicular** permite optimizar la operación de la infraestructura, reducir congestiones, mejorar la seguridad vial y proporcionar información valiosa para la planificación de mantenimiento y expansión de la red. Los datos recopilados son esenciales para la implementación de estrategias de gestión de demanda y tarificación dinámica.

## 8.5.3 Descripción Funcional y Técnica

### 3.1 Propósito del Sistema

El VDS/TDS tiene como propósito fundamental monitorear, contar y clasificar vehículos en tiempo real para proporcionar inteligencia operativa al centro de control. Sus objetivos específicos incluyen:

- **Medición de volumen**: Conteo preciso de vehículos por intervalos de tiempo.
- **Clasificación automática**: Categorización por tipo de vehículo (liviano, bus, camión, moto).
- **Cálculo de velocidad**: Determinación de velocidades puntual y media por tramo.
- **Detección de congestionamiento**: Identificación de condiciones de tráfico denso.
- **Generación de O/D**: Matrices origen-destino para análisis de patrones.

### 3.2 Tipos de Tecnologías de Detección

| **Tecnología** | **Principio de Funcionamiento** | **Aplicación Óptima** |
|----------------|--------------------------------|----------------------|
| **Bucles Inductivos** | Detección por cambio de inductancia al pasar un vehículo | Conteo y velocidad, alta precisión |
| **Radar Doppler** | Ondas de radio reflejadas por vehículos en movimiento | Velocidad y detección multicarril |
| **Lidar/Laser** | Escaneo láser 3D para perfilado de vehículos | Clasificación detallada y volumen |
| **Video Analytics (AI)** | Análisis de imágenes mediante inteligencia artificial | Clasificación visual y comportamiento |
| **Sensores Piezoeléctricos** | Señal eléctrica generada por presión del eje | Clasificación por ejes y peso estimado |
| **Sensores Infrarrojos** | Detección por interrupción del haz de luz IR | Conteo en múltiples carriles |

## 8.5.4 Justificación Estratégica

### 4.1 Importancia para la Operación Vial

El sistema de contadores de tráfico es fundamental para la gestión moderna de infraestructura por múltiples razones estratégicas:

1. **Visibilidad en Tiempo Real**: Proporciona al operador una visión completa del estado de la red vial, permitiendo respuestas proactivas ante incidentes.

2. **Optimización de Recursos**: Permite distribuir eficientemente recursos de mantenimiento y operación basándose en datos reales de uso.

3. **Gestión de Incidentes**: Detección temprana de congestionamientos y colisiones mediante análisis de patrones de velocidad.

4. **Planificación Estratégica**: Datos históricos para modelado de demanda y proyecciones de crecimiento.

## 8.5.11 Casos de Uso

### Caso de Uso 1: Gestión de Congestionamiento por Accidente

**Actores**: Operador de Control, Sistema VDS, Sistema VMS, Usuarios de la vía

**Descripción**: Un accidente múltiple reduce la capacidad de la vía, generando congestión en tiempo real.

**Flujo de Eventos**:
1. El sistema VDS detecta caída repentina de velocidad en el km 28+500 (de 100 km/h a 25 km/h).
2. Se activa alerta automática en el centro de control.
3. El operador verifica mediante CCTV la ocurrencia del accidente.
4. Se activan paneles VMS aguas arriba: "ACCIDENTE KM 28 - REDUZCA VELOCIDAD".
5. Los datos de VDS permiten estimar tiempo de demora (45 minutos).
6. Se informa a través de app móvil y redes sociales oficiales.
7. El operador monitorea la recuperación de velocidad para determinar fin del incidente.

**Resultado**: Reducción de congestión secundaria en 35%. Información oportuna a usuarios.

---

### Caso de Uso 2: Validación de Tarificación Dinámica

**Actores**: Sistema de Peaje, Sistema VDS, Sistema de Tarificación, Usuarios

**Descripción**: Implementación de tarifa diferenciada según nivel de congestión (congestion pricing).

**Flujo de Eventos**:
1. El sistema VDS mide flujo y velocidad en tiempo real en cada tramo.
2. Algoritmo de tarificación recibe datos cada 5 minutos.
3. Cuando velocidad cae bajo 40 km/h, se activa tarifa de hora pico.
4. Los datos de VDS validan que el vehículo transitó durante período de alta congestión.
5. Se aplica descuento automático si vehículo sale antes de hora pico.
6. Se generan reportes de correlación entre tarifa y cambio de patrones de viaje.

**Resultado**: Reducción del 22% en tráfico durante hora pico. Incremento en recaudo del 18%.

---

### Caso de Uso 3: Mantenimiento Predictivo de Vía

**Actores**: Equipo de Mantenimiento, Sistema VDS, Sistema de Gestión de Activos

**Descripción**: Uso de datos de tráfico para optimizar programación de mantenimiento vial.

**Flujo de Eventos**:
1. El sistema acumula datos históricos de tráfico (volumen, composición vehicular).
2. Algoritmo estima Equivalentes de Eje Estándar (EEE) por tramo.
3. Se identifican tramos con mayor carga estructural acumulada.
4. Se programa mantenimiento preventivo antes de falla estructural.
5. Se seleccionan horarios de menor flujo para minimizar impacto (02:00-05:00).
6. Se monitorea flujo post-mantenimiento para validar efectividad.

**Resultado**: Extensión de vida útil del pavimento en un 25%. Reducción de costos de emergencia.

---

### Caso de Uso 4: Detección de Incidentes por Patrón de Velocidad

**Actores**: Operador de Control, Sistema VDS, Sistema SOS, Servicios de Emergencia

**Descripción**: Identificación automática de incidentes mediante análisis de patrones de tráfico.

**Flujo de Eventos**:
1. VDS detecta patrón anómalo: velocidad alta seguida de parada súbita en un carril.
2. Algoritmo de incidentes calcula probabilidad de accidente (85%).
3. Se alerta al operador con ubicación exacta (km 45+320, carril derecho).
4. Operador verifica mediante CCTV y confirma accidente.
5. Se despacha automáticamente ambulancia y grúa.
6. Se cierra carril mediante señalización dinámica.
7. Se activan paneles VMS aguas arriba.

**Resultado**: Tiempo de detección reducido de 8 minutos a 45 segundos. Respuesta más rápida.

---

### Caso de Uso 5: Estudios de Origen-Destino (O/D) para Expansión

**Actores**: Departamento de Planeación, Sistema VDS, Consultor externo

**Descripción**: Uso de datos de conteo para planificar expansión de infraestructura.

**Flujo de Eventos**:
1. VDS recolecta datos continuamente por 12 meses en corredor existente.
2. Se analizan patrones horarios, diarios y estacionales.
3. Se identifican cuellos de botella recurrentes.
4. Se correlacionan datos con información de placa (sistema ALPR).
5. Se generan matrices O/D para diferentes períodos del día.
6. Se modela demanda futura considerando crecimiento urbano.
7. Se justifica inversión en tercer carril basado en datos.

**Resultado**: Decisión de inversión respaldada por datos. ROI proyectado positivo. Financiación aprobada.
