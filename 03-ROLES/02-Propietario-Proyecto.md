# 3.2 — Propietario del Proyecto (PP)

## Definición

El **Propietario del Proyecto (PP)** es el **cliente del proyecto**. Es la persona o entidad que:
- Tiene la necesidad que el proyecto debe satisfacer
- Es responsable del **éxito global del proyecto**
- Es propietario del resultado (producto/servicio) del proyecto

---

## Responsabilidades Clave

| Área | Responsabilidad |
|------|----------------|
| **Objetivos** | Establecer objetivos de negocio y criterios de éxito |
| **Caso de negocio** | Desarrollar y mantener el caso de negocio |
| **Autorización** | Autorizar el inicio del proyecto (Acta de Constitución) |
| **Recursos** | Garantizar disponibilidad de recursos y presupuesto |
| **Decisiones** | Presidir el CDP y tomar decisiones estratégicas |
| **Aceptación** | Aceptar entregables finales |
| **Éxito** | Ser responsable del éxito global del proyecto |

---

## Funciones en el CDP

- **Preside** el Comité de Dirección del Proyecto
- **Aprueba** los principales documentos de gestión:
  - Caso de negocio
  - Acta de constitución
  - Planes de gestión
  - Plan de implementación en el negocio
- **Decide** solicitudes de cambio significativas
- **Resuelve** incidencias elevadas al CDP

---

## Relación con otros Roles

| Rol | Relación |
|-----|----------|
| **DP** | Delega la gestión diaria, pero mantiene control estratégico |
| **RN** | Trabaja conjuntamente para asegurar valor de negocio |
| **PS** | Define necesidades, PS provee soluciones |
| **EDP** | No interactúa directamente, a través del DP |

---

## Perfil Ideal

- Conocimiento del negocio y contexto organizacional
- Capacidad de toma de decisiones
- Autoridad para asignar recursos
- Compromiso con el proyecto
- Visión estratégica
