# 2.2 — Fase 1: INICIO (Start)

## Objetivo

Definir los resultados deseados, crear el caso de negocio, definir el alcance, comenzar adecuadamente el proyecto.

---

## Flujo de la Fase de Inicio

```
┌─────────────────────────────────────────────────────────────────┐
│                     FASE 1: INICIO                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. REUNIÓN DE INICIO                                            │
│     └─ Alinear expectativas entre PP, RN, DP                    │
│                                                                  │
│  2. SOLICITUD DE INICIO DE PROYECTO                            │
│     └─ Formalizar la petición y necesidad                      │
│                                                                  │
│  3. CASO DE NEGOCIO                                              │
│     └─ Justificar la inversión: beneficios, costos, riesgos    │
│                                                                  │
│  4. ACTA DE CONSTITUCIÓN DEL PROYECTO                            │
│     └─ Autorizar formalmente el proyecto                       │
│                                                                  │
│  5. REVISIÓN LpP (Listo para Planificación)                      │
│     └─ Comité de Dirección aprueba pasar a Planificación       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Actividades Detalladas

### 1. Reunión de Inicio (Kick-off inicial)

**Participantes:** Propietario del Proyecto (PP), Responsable de Negocio (RN), futuro Director de Proyecto (DP)

**Objetivos:**
- Alinear expectativas sobre objetivos y alcance
- Identificar partes interesadas clave
- Evaluar viabilidad preliminar
- Decidir si procede formalizar la solicitud

**Resultado:** Acuerdo verbal o informal de proceder

---

### 2. Solicitud de Inicio de Proyecto

**Propósito:** Formalizar la petición de un nuevo proyecto

**Contenido:**
- Descripción de la necesidad
- Objetivos preliminares
- Partes interesadas identificadas
- Viabilidad preliminar

**Responsable:** Propietario del Proyecto (PP)

**Artefacto:** `OPM2-01.I.TPL` — Solicitud de Inicio de Proyecto

---

### 3. Caso de Negocio (Business Case)

**Propósito:** Justificar la inversión en el proyecto

**Secciones principales:**

| Sección | Contenido |
|---------|-----------|
| **1. Descripción de la necesidad** | ¿Qué problema oportunidad se aborda? |
| **2. Objetivos del proyecto** | ¿Qué se quiere lograr? (SMART) |
| **3. Alcance preliminar** | ¿Qué está incluido y qué no? |
| **4. Análisis de opciones** | ¿Qué alternativas se consideraron? |
| **5. Beneficios esperados** | Cuantificables e intangibles |
| **6. Estimación de costos** | Inversión total requerida |
| **7. Análisis de riesgos** | Riesgos principales y mitigaciones |
| **8. Cronograma preliminar** | Hitos clave y duración estimada |
| **9. Dependencias** | Relación con otros proyectos |
| **10. Recomendación** | ¿Proceder o no? |

**Responsable:** Propietario del Proyecto (PP) con apoyo del Responsable de Negocio (RN)

**Artefacto:** `OPM2-02.I.TPL` — Caso de Negocio

---

### 4. Acta de Constitución del Proyecto (Project Charter)

**Propósito:** Autorizar formalmente la existencia del proyecto y asignar al Director de Proyecto

**Secciones principales:**

| Sección | Contenido |
|---------|-----------|
| **1. Propósito del proyecto** | Justificación y objetivos |
| **2. Descripción del proyecto** | Alcance, entregables principales |
| **3. Requisitos principales** | Necesidades clave de las partes interesadas |
| **4. Límites del proyecto** | Lo que está fuera de alcance |
| **5. Asignación de recursos** | Presupuesto preliminar |
| **6. Cronograma preliminar** | Fechas clave |
| **7. Riesgos iniciales** | Principales amenazas |
| **8. Criterios de éxito** | ¿Cómo se medirá el éxito? |
| **9. Roles clave** | PP, RN, DP asignados |
| **10. Autorización** | Firma del PP y patrocinador |

**Responsable:** Propietario del Proyecto (PP)

**Artefacto:** `OPM2-03.I.TPL` — Acta de Constitución del Proyecto

## Punto de Revisión: LpP (Listo para Planificación)

### Propósito
Asegurar que el proyecto está debidamente justificado y autorizado antes de invertir recursos en planificación detallada.

### Criterios de Evaluación

| Criterio | Pregunta |
|----------|----------|
| Justificación | ¿El caso de negocio es convincente? |
| Autorización | ¿El acta de constitución está firmada? |
| Recursos | ¿Hay presupuesto preliminar aprobado? |
| Viabilidad | ¿Los riesgos iniciales son aceptables? |
| Roles | ¿Los roles clave están asignados? |

### Decisiones Posibles

| Decisión | Significado |
|----------|-------------|
| **APROBAR** | Proceder a la fase de Planificación |
| **CONDICIONAR** | Proceder con ajustes identificados |
| **RECHAZAR** | No proceder, documentar razones |
| **APLAZAR** | Posponer decisión hasta obtener más información |

### Quién Decide
El **Comité de Dirección del Proyecto (CDP)**, presidido por el Propietario del Proyecto (PP).

## Duración y Esfuerzo

| Perfil de Proyecto | Duración típica | Esfuerzo |
|-------------------|-----------------|----------|
| **Light** | 3-5 días | 20-40 horas |
| **Medium** | 2-3 semanas | 80-120 horas |
| **Heavy** | 4-6 semanas | 160-240 horas |

---

## Riesgos Comunes en la Fase de Inicio

| Riesgo | Impacto | Mitigación |
|--------|---------|------------|
| Alcance ambiguo | Desviaciones costosas | Definir explícitamente límites |
| Estimaciones optimistas | Presupuesto insuficiente | Usar estimación por tres valores |
| Stakeholders ignorados | Resistencia posterior | Identificar a todos los afectados |
| Caso de negocio débil | Proyecto sin valor | Revisar rigurosamente beneficios |
| Presión por empezar | Saltarse planificación | Resistir hasta tener LpP aprobado |
