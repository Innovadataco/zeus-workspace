# 2.4 — Fase 3: EJECUCIÓN (Execute)

## Objetivo

Coordinar la ejecución de los planes, producir entregables, gestionar el trabajo diario, mantener comunicación con partes interesadas.

---


## Actividades Detalladas

### 1. Reunión Inicial de Ejecución (Kick-off)

**Participantes:** DP, RN, EDP completo, PS si aplica

**Objetivos:**
- Presentar manual del proyecto y planes al equipo
- Confirmar roles y responsabilidades
- Establecer ritmo de trabajo (reuniones, informes)
- Motivar al equipo
- Resolver dudas iniciales

---

### 2. Coordinación del Proyecto

**Responsable:** Director de Proyecto (DP)

**Tareas:**
- Dirigir reuniones de seguimiento (semanales/bisemanales)
- Resolver conflictos y impedimentos
- Gestionar dependencias entre actividades
- Mantener actualizado el plan de trabajo
- Gestionar relaciones con partes interesadas

---

### 3. Producción de Entregables

**Responsable:** Equipo de Desarrollo del Proyecto (EDP)

**Enfoque:**
- Desarrollar según especificaciones del alcance
- Seguir estándares de calidad definidos
- Documentar avances en herramientas de seguimiento
- Solicitar revisiones intermedias

**Para proyectos ágiles:** Iteraciones de 1-4 semanas con entregables parciales

---

### 4. Aseguramiento de Calidad

**Responsable:** DP + Responsable de Calidad (si existe)

**Actividades:**
- Revisar entregables contra criterios de aceptación
- Realizar pruebas y validaciones
- Documentar no conformidades
- Implementar acciones correctivas

---

### 5. Informes del Proyecto

**Tipos de Informes:**

| Informe | Frecuencia | Audiencia | Artefacto |
|---------|-----------|-----------|-----------|
| **Informe de progreso** | Semanal/Mensual | CDP, PP | OPM2-22 |
| **Informe de situación** | Mensual/Trimestral | Stakeholders | OPM2-19 |
| **Informe de situación (GVG)** | Cuando sea necesario | Alta dirección | OPM2-20 |
| **Informe de fin de proyecto** | Al cierre | CDP | OPM2-33 |

**Contenido típico:**
- Resumen del estado (semáforo: verde/amarillo/rojo)
- Avance contra cronograma
- Avance contra presupuesto
- Riesgos principales y mitigaciones
- Incidencias abiertas
- Decisiones requeridas
- Próximos hitos

---

### 6. Gestión de Cambios

**Flujo:**

```
Solicitud de Cambio → Análisis de Impacto → Decisión (CDP) → Implementación → Verificación
```

**Artefactos:**
- `OPM2-21` — Formulario de Solicitud de Cambio
- `OPM2-26` — Registro de Cambios

---

### 7. Gestión de Riesgos

**Ciclo continuo:**

```
Identificar → Evaluar (Probabilidad × Impacto) → Planificar respuesta → Implementar → Monitorear
```

**Artefacto:** `OPM2-23` — Registro de Riesgos

---

### 8. Gestión de Incidencias

**Flujo:**

```
Reportar → Evaluar → Asignar → Resolver → Verificar → Cerrar
```

**Artefacto:** `OPM2-24` — Registro de Incidencias

## Punto de Revisión: LpC (Listo para Cierre)

### Propósito
Asegurar que todos los entregables están completos, aceptados y el proyecto está listo para cerrar formalmente.

### Criterios de Evaluación

| Criterio | Pregunta |
|----------|----------|
| Entregables | ¿Todos los entregables están completos y aceptados? |
| Calidad | ¿Los entregables cumplen los criterios de calidad? |
| Requisitos | ¿Todos los requisitos están satisfechos? |
| Documentación | ¿La documentación está completa? |
| Lecciones | ¿Se han capturado lecciones aprendidas? |
| Transición | ¿Se ha planificado la transición a operaciones? |

### Decisiones Posibles

| Decisión | Significado |
|----------|-------------|
| **APROBAR** | Proceder a la fase de Cierre |
| **CONDICIONAR** | Completar acciones pendientes antes de cerrar |
| **RECHAZAR** | Continuar ejecución hasta cumplir criterios |
| **APLAZAR** | Posponer decisión |

---

## Duración

| Perfil de Proyecto | Duración típica |
|-------------------|-----------------|
| **Light** | 1-3 meses |
| **Medium** | 3-9 meses |
| **Heavy** | 9-24 meses (o más) |

---

## Checklist de Salida de Fase

- [ ] Todos los entregables completos
- [ ] Aceptación formal de entregables por PP/RN
- [ ] Registro de riesgos actualizado (sin riesgos críticos abiertos)
- [ ] Registro de incidencias resuelto (sin incidencias críticas)
- [ ] Lecciones aprendidas capturadas
- [ ] Informe de progreso final emitido
- [ ] Plan de transición aprobado
- [ ] Reunión LpC realizada
- [ ] Decisión documentada
